# MVP 开发任务拆解

## MVP 目标

交付一个可运行的 Skill 蒸馏器，能够从人物 / 主题 / 模糊需求出发，完成：

- 目标 Skill 目录初始化
- `6` 个并行调研 Agent
- `1` 个交叉验证 Agent
- 研究文件落盘
- 认知框架提炼
- 最终 `SKILL.md` 生成
- 基本质量验证

## Workstream 1 — Skill 脚手架

### 目标
保证每次运行都能快速生成标准目录。

### 任务
1. 创建目标 Skill slug 规则
2. 建立 `templates/target-skill/` 目录模板
3. 生成 `01-07` research 文件骨架
4. 生成目标 `SKILL.md` 与 `validation/validation-report.md` 模板
5. 支持已有 Skill 的增量模式检测

### 完成标准
- 一次模板复制与占位替换即可生成完整目录
- 不覆盖已有文件，除非明确进入增量更新或重建
- 所有 research 文件都具备统一标题结构

## Workstream 2 — Phase 1 并行调研编排

### 目标
稳定运行 `6` 个单维度 Agent。

### 任务
1. 固化 `6` 个 Agent 的职责边界
2. 固化每个 Agent 的输出文件路径
3. 固化来源标注与评分规则
4. 约束 Agent 只输出发现 / 证据 / 冲突 / 缺口
5. 规定失败兜底：失败即标记信息不足，不阻塞主流程

### 完成标准
- 任意人物 / 主题都能跑出 `01-06` 文件
- 每个文件都能看出来源、结论、缺口
- 没有 Agent 越权做最终合成

## Workstream 3 — Phase 1.5 交叉验证与补强

### 目标
在生成最终 Skill 之前，先把冲突和薄弱点暴露出来。

### 任务
1. 定义 Agent 7 读取 `01-06`
2. 输出冲突、关系网络、可信度调整
3. 产出第二轮补强建议
4. 定义薄弱维度触发条件
5. 允许按需进入第二轮调研

### 完成标准
- `07-cross-validation.md` 必然生成
- 能指出至少一轮需要补强的维度或确认可直接进入提炼

## Workstream 4 — Phase 2 提炼与构建

### 目标
把研究结果提炼成真正可运行的目标 Skill。

### 任务
1. 提炼 `3-7` 个心智模型
2. 提炼 `5-10` 条决策启发式
3. 提炼表达 DNA
4. 写出价值观、反模式、智识谱系、诚实边界
5. 生成最终目标 `SKILL.md`

### 完成标准
- 最终 Skill 不是资料堆砌，而是可执行框架
- 每个模型有证据与失效条件
- 主题型产出保持专业中性，人物型产出保留辨识度

## Workstream 5 — Phase 4 验证与交付

### 目标
保证 Skill 可用，不是看起来完整。

### 任务
1. 已知问题一致性测试
2. 边缘问题推断测试
3. 风格辨识度测试
4. 按 `references/validation-contract.md` 生成 `validation/validation-report.md`
5. 输出交付摘要并写明局限和调研截止时间

### 完成标准
- 至少完成 `3 + 1 + 1` 组测试
- 验证报告已落盘且结论明确
- 若不过关，可回到提炼阶段最多迭代 `2` 轮
- 局限写清楚，不假装完美

## MVP 外的增强项

以下内容不阻塞 MVP：

- 自动二轮任务调度器
- 多语言材料自动配比
- 视频 / 播客转录自动化
- 关系网络可视化
- 自动打包发布
- 更细的增量更新策略

## 推荐实施顺序

1. 先完成脚手架
2. 再固化 `6+1` Agent 协议
3. 再完成提炼与最终 Skill 生成
4. 最后补验证与增强项
